# AGENTS.md

## Development Flow

이 저장소의 기능 개발은 아래 순서를 기본 플로우로 사용한다.

1. 원하는 기능을 구현한다.
2. 실제 `.hwpx` 산출물을 만든다.
3. `Hancom Office HWP Viewer`로 PDF 인쇄를 수행한다.
4. 인쇄된 PDF 결과를 확인한다.
5. 문제가 없으면 개발 완료로 본다.
6. 문제가 있으면 확인된 피드백을 기준으로 수정 후 다시 같은 플로우로 검증한다.

## Verification Rule

- 구조 검증이나 단위 테스트만으로 완료 처리하지 않는다.
- 최종 완료 기준은 `Hancom Office HWP Viewer` 인쇄 결과에서 기대한 렌더링이 확인되는 것이다.
- 검증 산출물은 가능하면 `output/` 아래에 남겨 before/after 비교가 가능하게 한다.

## Notes

- PDF 인쇄 검증이 필요한 기능은 구현 직후 바로 수동 검증까지 이어서 진행한다.
- 렌더링 이슈가 발견되면 원인 분석, 수정, 동일 방식 재검증을 한 세트로 처리한다.
